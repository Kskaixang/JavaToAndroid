# 🚀 Auto SSH Tunnel 開發者須知與原理解析

這份文件旨在向未來的接手者（或忘記細節的你）解釋這套「**全自動 SSH 通道機制**」是如何運作的，以及新進開發者該如何無痛上手。

---

## 🛑 為什麼我們需要這個機制？
基於**資安最高原則**，我們雲端 Oracle 伺服器上的 MySQL (Port 3306) **「絕對禁止」**暴露在公網上，否則會遭到機器人無限的密碼暴力破解。
因此，若要在自己的電腦 (本機) 寫程式並連接雲端資料庫，唯一的合法途徑就是建立 **SSH 通道 (SSH Tunnel)**。

傳統作法是：開發者每次寫程式前，都要手動打開終端機輸入落落長的 `ssh -L ...` 指令。這不僅煩人，還容易出錯。
為了達到微服務的「極致開發體驗 (DX)」，我們把這個過程**100% 全自動化**了！

---

## ⚙️ 它是怎麼運作的？(底層魔法)

我們在專案中撰寫了一支 `SshTunnelInitializer.java`，並把它強制綁定在 Spring Boot 的啟動最前端 (`SpringToAndroidApplication.java` 的 `main` 函式中)。

當你按下 IntelliJ 的綠色 Run 啟動專案時，會觸發以下連鎖反應：

1. **環境探測**：它會先去偵測你的作業系統。如果是 Linux (代表已經部署在 Oracle 雲端上了)，它就會乖乖閉嘴，什麼都不做。**只有在 Windows (代表本機開發機) 才會觸發。**
2. **防呆檢查**：它會下指令檢查本機的 `3307` Port 是不是已經在用了？如果是，代表通道早就建好了，直接跳過。
3. **智慧金鑰彈窗**：它會去你的家目錄尋找隱藏的設定檔。如果找不到，它會臨時呼叫 Windows 原生的檔案選擇視窗，請你指定 `.key` 金鑰在哪裡。
4. **絕對隔離的快取**：你選完金鑰後，它會把你選的路徑存入 `C:\Users\你的名字\.springtoandroid_ssh_key`。
   > [!TIP]
   > 為什麼要存在這裡？因為家目錄「**不在 Git 的管轄範圍內**」！這保證了每個開發者的本機路徑都不一樣，而且金鑰路徑永遠不會被外洩到 GitHub 上。
5. **背景啟動與優雅關閉**：拿到路徑後，它在背景默默幫你下好 SSH 指令，等你一秒鐘，然後放行讓 Spring Boot 的資料庫連線 (HikariCP) 順利通關。當你在 IntelliJ 按下紅色的 Stop 停止專案時，它觸發「Shutdown Hook」，把背景的 SSH 通道清得乾乾淨淨。

## 👨‍💻 新進開發者 (後人) 上手指南

如果你是剛 Clone 這份專案的新人，請按照以下步驟，只要做一次，終身受用：

1. **準備金鑰**：向團隊取得 Oracle 雲端的 SSH 私鑰 (`.key` 或 `.pem` 檔)，隨便放在你電腦裡安全的地方 (例如 `C:\` 或文件夾)。
2. **設定環境變數**：在 IntelliJ 的 Run Configuration 中，補上環境變數 `DB_PASSWORD=資料庫的密碼`。
3. **啟動專案**：直接點擊 IntelliJ 的 Run。
4. **選擇金鑰**：此時畫面會彈出一個視窗「請選擇 Oracle Cloud 的 SSH 金鑰檔案」，選擇你剛剛存放的金鑰。
5. **大功告成**：視窗關閉後，專案就會瞬間連上雲端資料庫並啟動完畢！

### 💡 常見問答 (FAQ)
**Q: 如果我的金鑰換位子了，或者想換一把金鑰怎麼辦？**
A: 非常簡單！只要去你的家目錄 (`C:\Users\你的名字\`)，把一個名叫 `.springtoandroid_ssh_key` 的隱藏檔案**刪除**。下次啟動 Spring Boot 時，它就會重新彈出視窗讓你重選一次了。

**Q: 為什麼 application.properties 的 URL 是寫 3307？**
A: 因為 `SshTunnelInitializer` 在背景把雲端的 3306 牽線到了你本機的 3307。所以 Spring Boot 連線本機 3307，其實就是在連雲端！
