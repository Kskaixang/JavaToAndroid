# 導入 Google 登入與 JWT 認證機制

本計畫旨在為 `JavaToAndroid` 專案實作前端 (Android) 與後端 (Spring Boot) 的 Google 登入流程，並透過 JWT (JSON Web Token) 進行 API 請求的安全防護。本機制參考了 `forAI` 專案的雙 ID 驗證流程，且均使用 Java 語言進行實作。

## User Review Required

> [!IMPORTANT]
> 1. **憑證需求**: 您需要進入 Google Cloud Console 申請一個「Android Client ID」與一個「Web Client ID」，且 Android Client ID 必須綁定您開發環境的 `debug.keystore` SHA-1 指紋。
> 2. **本機設定檔**: 您需要在 Android 的 `local.properties` 內新增 `is.local.test=true` (若無) 以及新的 `web.client.id="您的WebClientID"`，這部分涉及隱私不會寫死在 Git 中。
> 3. **開發語言**: 經檢查，目前的 `JavaToAndroid` 前端完全採用 Java 開發 (與參考的 Kotlin Multiplatform 不同)，因此所有 Android 端實作將使用 Java。

## Open Questions

> [!WARNING]
> 1. **User Entity 設計**: 後端目前還沒有 `User` 表，計畫中將建立包含 `googleId`, `email`, `name`, `avatarUrl` 等基本欄位的 `User` 實體。這符合您的需求嗎？
> 2. **Token Storage**: 前端打算使用 Android 基礎的 `SharedPreferences` (或 `EncryptedSharedPreferences`) 來儲存 JWT。是否有其他特定的儲存偏好？

## Proposed Changes

### Backend (SpringToAndroid)

#### [MODIFY] [pom.xml](file:///c:/workspace/JavaToAndroid/SpringToAndroid/pom.xml)
- 新增 `spring-boot-starter-security` (安全認證模組)
- 新增 `jjwt-api`, `jjwt-impl`, `jjwt-jackson` (產生與解析 JWT)
- 新增 `google-api-client` (驗證 Google 發行的 ID Token)

#### [MODIFY] [application.properties](file:///c:/workspace/JavaToAndroid/SpringToAndroid/src/main/resources/application.properties)
- 新增 JWT 的 Secret Key 與 Expiration (過期時間) 設定。
- 新增 Google 的 Web Client ID 設定。

#### [NEW] User & Repository
- `User.java` (實體類別，對應 MySQL/H2 資料表)
- `UserRepository.java` (JPA 存取介面)

#### [NEW] Security & JWT
- `JwtUtils.java`: 負責簽發系統專屬 JWT 與解析 Token。
- `JwtRequestFilter.java`: 攔截所有請求，檢查 Header 中的 `Authorization: Bearer <Token>`。
- `SecurityConfig.java`: 設定 Spring Security，放行 `/api/auth/google-login`，其餘 API 均須經過身分驗證。

#### [NEW] Auth API
- `AuthService.java`: 負責接收 Android 傳來的 Google Token，使用 Google API Client 驗證真偽。驗證成功後查找或建立 User，並回傳 App JWT。
- `AuthController.java`: 暴露出 `POST /api/auth/google-login` 端點。

---

### Frontend (Android App)

#### [MODIFY] [build.gradle](file:///c:/workspace/JavaToAndroid/app/build.gradle)
- 新增 `com.google.android.gms:play-services-auth` (Google Sign-in SDK)。
- 修改 `defaultConfig` 從 `local.properties` 讀取並注入 `WEB_CLIENT_ID` (透過 `buildConfigField`)。

#### [NEW] Auth API & Storage
- `AuthApi.java`: 增加 `POST /api/auth/google-login` 的 Retrofit 介面。
- `TokenStorage.java`: 將後端回傳的 JWT 儲存至 `SharedPreferences`。

#### [NEW] Http Interceptor
- `AuthInterceptor.java`: 實作 OkHttp 的 `Interceptor`，每次發請求前自動把 `TokenStorage` 裡的 JWT 取出，並加入 HTTP Header 中。
- `RetrofitClient.java` (修改): 將 `AuthInterceptor` 綁定到 Retrofit 的 OkHttpClient 上。

#### [NEW] Google Login Logic & UI
- `GoogleSignInHelper.java`: 封裝 Google SDK 的 `GoogleSignInOptions` 與 Intent 發起動作。
- `AuthRepository.java`: 負責協調「呼叫 Google 登入 -> 拿 Token 打後端 -> 儲存 JWT」的業務流程。
- `LoginActivity.java`: 提供一個包含 "Sign in with Google" 按鈕的登入頁面作為進入點。

## Verification Plan

### Automated Tests
無自動化測試。將進行手動測試。

### Manual Verification
1. 在 Android 端點擊「Google 登入」。
2. 彈出 Google 帳號選擇視窗，選擇帳號後成功獲取 Google Token。
3. App 發送 Token 至 Spring Boot 後端，後端印出解析結果並成功簽發 JWT 給 App。
4. App 攜帶此 JWT 嘗試呼叫一個受保護的後端 API (例如取得股票或隨機單字)，確認能成功獲得資料；若不帶 Token 則回傳 401 Unauthorized 或 403 Forbidden。
